## 🎯 "The Nexus"가 뭐예요?

### 초등학생도 이해하는 설명

**The Nexus** = "우리만의 똑똑한 도우미들이 사는 집"

상상해보세요:
- **집 (House)**: 우리 컴퓨터
- **도우미들 (Agents)**: Sudal(의장), Lando(관리자), OpenClaude(개발자) 등
- **도우미들이 사용하는 것들 (Tools)**:
  - 📚 기억 창고 (Knowledge Graph) = 지금까지 배운 것들 저장
  - 💬 팀 채팅방 (Discord Bridge) = 팀원들과 소통
  - 🧠 생각하는 뇌 (LLM via Ollama) = 로컬 AI

**핵심**: 다른 회사의 AI(ChatGPT, Claude 등)를 안 쓰고, **우리가 직접 우리 컴퓨터에서 모든 걸 관리**한다!

### 왜 이렇게 만들까?

| 문제 | 해결책 |
|:---|:---|
| OpenAI에 의존 → 비용 증가, 정책 변경 위험 | 우리가 Ollama로 로컬 LLM 실행 |
| 개인정보 Google/OpenAI 서버에 저장 | 모든 데이터가 우리 컴퓨터에만 있음 |
| 인터넷 끊기면 못 씀 | 로컬에서 완전 작동 가능 |

## 🏗️ "The Nexus" 아키텍처 (아주 쉽게)

```
┌─────────────────────────────────────────┐
│         화면 (우리가 클릭하는 곳)          │
│    Electron App + Vue 3 (UI)            │
│  - 에이전트 조종판                       │
│  - 기억 창고 지도 보기                   │
│  - 채팅 창                              │
└────────────┬────────────────────────────┘
             │
        IPC 케이블
        (안전한 통로)
             │
┌────────────▼────────────────────────────┐
│       뒷방 (눈에 안 보이는 곳)            │
│    Node.js Server                       │
├─────────────────────────────────────────┤
│                                         │
│  ┌─ Ollama Manager                     │
│  │   (로컬 AI 뇌 관리)                  │
│  │                                     │
│  ├─ LLM Agent System                   │
│  │   (도우미들이 생각하고 행동)         │
│  │                                     │
│  ├─ Knowledge Graph                    │
│  │   (기억 창고)                       │
│  │                                     │
│  ├─ Discord Bridge                     │
│  │   (팀 채팅 연결)                    │
│  │                                     │
│  └─ Config Manager                     │
│      (설정 기억)                        │
│                                         │
└─────────────────────────────────────────┘
```

**이게 왜 중요한가?**
- 화면(UI)과 뒷방(로직)이 분리됨 → 하나가 깨져도 다른 하나는 안 깨짐
- IPC 케이블이 안전한 통로 역할 → 악성 코드가 우리 컴퓨터 파일에 접근 못함
- 각 부분(모듈)이 독립적 → 필요하면 클라우드로 옮기기 쉬움

## 📅 "언제까지 뭘 할 거예요?"

### Phase 1: Zero Cost (무료로 완성) - 2026-05-20 ~ 2026-06-30

#### 목표: "우리 컴퓨터에서 모든 게 작동하게!"

**왜 Zero Cost?**
- Ollama (무료) + Node.js (무료) + Electron (무료) = 돈 안 씀
- 클라우드 AI 서비스 = 안 씀

**뭘 만들까?**

**1-1. 기본 부품들 (이미 완료)**
- ✅ Ollama 매니저 (로컬 AI 관리)
- ✅ 에이전트 시스템 (3개 도우미: Sudal, Lando, OpenClaude)
- ✅ 기억 창고 (Knowledge Graph)
- ✅ Discord 연결 (팀 채팅)
- ✅ 설정 관리자 (모든 설정 기억)
- ✅ IPC 통로 (UI↔뒷방 안전 통신)

**1-2. 화면 만들기 (진행 중: 2026-05-20 ~ 2026-05-31)**

| 태스크 | 뭐 하는 거? | 왜 필요? | 언제까지? |
|:---:|:---|:---|:---:|
| **Ollama 모델 받기 UI** | 사용자가 클릭 한 번으로 AI 뇌 자동 설치 | 없으면 사용자가 터미널 명령어를 쳐야 함 (어려움) | 2026-05-24 |
| **기억 창고 지도 보기** | 배운 정보들을 시각적으로 보여줌 | 글자만으로는 관계를 이해하기 어려움 | 2026-05-28 |
| **에이전트 조종판** | 도우미들에게 일 시키기 | 프로그래머가 아닌 일반인도 쓸 수 있게 | 2026-05-31 |
| **Discord 설정 마법사** | 클릭 몇 번으로 Discord 연결 | 터미널 안 쓰고 GUI에서 쉽게 | 2026-05-29 |

**1-3. 테스트 & 문서 (2026-06-01 ~ 2026-06-15)**

| 뭐? | 왜? | 어떻게? |
|:---|:---|:---|
| **단위 테스트 20개+** | 각 부품이 혼자서 잘 작동하는지 확인 | 자동화된 테스트 코드 |
| **통합 테스트 3개** | 부품들을 모아서 함께 잘 작동하는지 확인 | 실제 사용 시나리오로 테스트 |
| **사용자 설명서** | 일반인도 쓸 수 있게 | 스크린샷과 함께 단계별 가이드 |

**1-4. CDC 대시보드 연결 (2026-06-16 ~ 2026-06-30)**

**뭐하는 거?**
- The Nexus 뒷방(Node.js)이 제공하는 기능들을 CDC 웹페이지에서 사용할 수 있게 연결

**왜?**
- CDC = 관리자 대시보드 (웹)
- The Nexus = AI 뇌와 기억 창고 (데스크톱)
- 둘을 연결하면 → 웹에서 AI 도우미들을 조종 가능

**어떻게?**
- REST API 또는 npm 패키지로 연결 (나중에 정함)
