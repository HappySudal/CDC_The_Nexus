## 📄 각 파일의 역할

### 01_DevelopPlan_The_Nexus.md
- **목적**: The Nexus의 기술 아키텍처 + 구현 상세 설계
- **작성**: Phase 설계 시 (초기 작성 후 대체로 변경 없음)
- **업데이트 빈도**: 매우 드물게 (설계 변경 시만)
- **권한**: Claude (Opus) - 변경 전 의장님 승인 필수
- **역할**: 개발팀의 "설계 가이드북"

**금지 사항**:
- ❌ 진행 상황을 이 파일에 기록
- ❌ Task별 완료 여부를 이 파일에 기록
- ❌ 날짜별 업데이트 (타임스탬프 불필요)

---

### 02_ProgressManagementPlan_The_Nexus.md
- **목적**: 일정, 위험, 품질 관리
- **작성**: 프로젝트 시작 시 초기화
- **업데이트 빈도**: 주 1-2회 (또는 주요 변화 시)
- **권한**: Claude (자동 업데이트) + 의장님 검증
- **역할**: 의장님의 "공정 감시 대시보드"

**업데이트 항목**:
- ✅ 일정 편차 (실제 vs 예상)
- ✅ 위험 요소 (발생 여부, 해결 상태)
- ✅ 품질 메트릭 (달성도)
- ✅ 변경 로그 (구조 변화)

**형식**:
```
| 항목 | 예상 | 실제 | 편차 | 상태 |
|:---:|:---:|:---:|:---:|:---|
| Week 1 | 100% | [실제값]% | [편차] | [상태] |
```

---

### 03_CheckList_The_Nexus.md
- **목적**: 실시간 Task 진행 상황 추적
- **작성**: Phase 1 시작 시 초기화
- **업데이트 빈도**: 매일 또는 Task 완료 시마다
- **권한**: Claude (자동) + Antigravity (실행 담당)
- **역할**: 의장님의 "실시간 진행률 보기"

**업데이트 항목**:
- ✅ Task 상태 (☐ 대기 / ⏳ 진행 / ☑️ 완료)
- ✅ 진행률 (%)
- ✅ 완료 여부
- ✅ 예상 완료일

**형식**:
```
### 1-1. Task 이름
- [✅] 소항목 1
- [⏳] 소항목 2
- [ ] 소항목 3
**상태**: ⏳ 진행 중
**진행률**: 50%
**담당**: Claude / Antigravity
**예상 완료**: Day 2
```

## 📅 업데이트 일정

### Daily (매일)
```
시간: 18:00 KST
담당: Claude
작업:
  - 03_CheckList 업데이트 (Task 완료 상황)
  - 진행률 계산
  - 의장님에게 자동 알림
```

### Weekly (주 1-2회)
```
시간: 월요일 10:00 / 금요일 18:00
담당: Claude
작업:
  - 02_ProgressManagementPlan 전체 업데이트
  - 위험 요소 재평가
  - 일정 편차 분석
  - 의장님께 주간 리포트
```

### Monthly (필요시)
```
담당: Claude + 의장님
작업:
  - 전체 진행도 평가
  - 구조적 변경 필요성 검토
  - 다음 달 계획 조정
```

## 🔔 의장님 보고 규칙

### 매일 보고 (Daily Report)
```
형식: 간단 메시지
내용:
  - 어제 완료: Task X 100%
  - 오늘 진행: Task Y (진행률%)
  - 주요 이슈: (있으면 보고)

예시:
"✅ Task 1-1 완료 (100%)
 ⏳ Task 1-2 진행 중 (50%)
 주요: 없음"
```

### 주간 보고 (Weekly Report)
```
형식: 상세 표
내용:
  - Phase X 진행률: Y%
  - 완료 Task: (개수)
  - 위험 상태: (발생한 위험)
  - 다음주 계획: (예정 Task)

장소: Discord 또는 The Nexus 대시보드
```

### 긴급 보고 (Emergency Report)
```
조건: 위험이 발생했을 때
시간: 즉시
형식: Task 중단 + 원인 설명 + 대응 방안
```

✅ 매일 정시 업데이트 (18:00)  
✅ 주간 상세 분석 (월/금)  
✅ 모든 변경사항 기록  
✅ 타임스탐프 명시 (CheckList 제외, Progress는 필수)

## 📊 누구든지 따라할 수 있는 사용 가이드

### **Case 1: 의장님이 하루를 시작할 때**
```
1️⃣ 폴더 열기: 02_DevelopPlan/
2️⃣ 파일 순서대로 읽기:
   - 01_DevelopPlan_The_Nexus.md (기술 설계)
   - 02_ProgressManagementPlan_The_Nexus.md (위험/일정)
   - 03_CheckList_The_Nexus.md (오늘 진행도)
3️⃣ 필요시 참고:
   - 04_ValidationReport.html (검증 결과)
   - 05_FinalDeploymentReport.html (배포 상태)
4️⃣ 필요한 지시 내리기

= 총 5-10분 내에 전체 상황 파악 가능
```

### **Case 2: 새로운 팀원이 프로젝트에 합류할 때**
```
1️⃣ 01_DevelopPlan_The_Nexus.md 읽기 (전체 기술 이해)
2️⃣ 02_ProgressManagementPlan_The_Nexus.md 읽기 (현재 상황)
3️⃣ 03_CheckList_The_Nexus.md 읽기 (할 일 파악)
4️⃣ 본인의 Task 시작

= 총 30-40분 온보딩 완료
```

### **Case 3: Claude/Antigravity가 매일 아침 업무 시작할 때**
```
1️⃣ 03_CheckList_The_Nexus.md 읽기 (오늘의 Task 확인)
2️⃣ 02_ProgressManagementPlan_The_Nexus.md 업데이트 (위험/일정)
3️⃣ 의장님에게 Daily Report 전송
4️⃣ 할당된 Task 실행

= 총 5분 내에 시작 가능
```
