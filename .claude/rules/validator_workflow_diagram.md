## 🔄 Loop 상세 흐름

```
╔════════════════════════════════════════════════════════════════════════════╗
║                    Loop 체계 (최대 3회)                                    ║
╚════════════════════════════════════════════════════════════════════════════╝

Loop 1
──────────────────────────────────────────────────────────────────────────────
│
├─ Validator 검증 완료 → 점수 산정
│  └─ 예: 75/100, Conditional Pass
│
├─ QA_Report 작성
│  └─ 개선 권고사항:
│     ① Security: SQL injection 3건 수정
│     ② Cross-System: 2개 파일 경로 동기화
│     ③ Code Quality: 기술부채 2건 리팩토링
│
├─ Executor에 Loop 지시 전송
│
└─ 대기: Executor 재개발 (24-48시간)


Loop 2
──────────────────────────────────────────────────────────────────────────────
│
├─ Executor 완료 신호 → 재제출
│
├─ Validator 재검증 (STEP 4 반복)
│  └─ 점수: 88/100, PASS ✅
│
├─ 또는 점수: 72/100, Conditional Pass ⚠️
│
└─ 대기: Executor 재개발 또는 Loop 3 진행


Loop 3 (최종)
──────────────────────────────────────────────────────────────────────────────
│
├─ Executor 최종 개선 제출
│
├─ Validator 최종 검증
│  └─ 점수 ≥70점 → Conditional Pass 강제 승인
│  └─ 점수 <70점 → FAIL (프로젝트 보류)
│
└─ 다음 단계: STEP 5 또는 의장님 판단


조기 종료 조건
──────────────────────────────────────────────────────────────────────────────
1️⃣  Loop 중 새로운 Critical 발견 → 즉시 FAIL (재분석)
2️⃣  누적 토큰 120K 초과 → 자동 중단 (의장님 판단)
3️⃣  3회 Loop 완료 후 70점 이상 → Conditional Pass 자동 승인
```

## 🎓 학습 포인트

| 개념 | 의미 | 예시 |
|:---|:---|:---|
| **독립성** | Validator은 Executor 의견 무시 | "테스트는 통과했지만 코드 구조가..." |
| **객관성** | 증거 기반 판정만 | 감정적 평가 금지 |
